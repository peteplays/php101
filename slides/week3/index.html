  "one" => "one",

    "sub_one" => "sub_one",
    "sub_two" => 2,
    "sub_three" => array(1, 2, 3)
  )

An associative aray makes use of key => value pairs

Some languages have different constructs for different types of arrays, but PHP uses only one

Working with associative arrays


<?php
$car = array(2014, "black", 4);
$car = array(
  "year" => 2012,
  "color" => "black",
  "doors" => 4
);

$car["make"] = "BMW";
unset($car["year"]);

array_key_exists("year", $car); // false
array_key_exists("doors", $car); // true

foreach ($car as $key => $value) {
  echo $key . ": " . $value;
  echo $car[$key];
}

foreach ($car as $value) {
  echo $value;
}

for ($i = 0; $i < count($car); $i++) {
  echo $car[$i]; // undefined offset: 0/1/2
}
?>

Just like normal arrays, associative arrays can be manipulated the same way

array key exists good for checking when you don't know if a key will be there when iterating over an array

You can only iterate over associative arrays with a foreach statement

Object Oriented Programming


<?php
public class Bicycle
{
  public $size;
  public $chain;
  public $tireSize;

  public function __construct($size, $chain, $tireSize)
  {
    $this->size = $size;
    $this->chain = $chain;
    $this->tireSize = $tireSize;
  }

  public function tireSize()
  {
    return "The tire size is " . $this->tireSize . " inches";
  }
}

$bike = new Bike("26", "10-speed", "23");
echo $bike->tireSize(); // 23
echo $bike->chain; // 10-speed
?>

Magic Methods


<?php
__construct();
__destruct();
?>

__construct is used for initialization

__destruct is used for cleaning up or releasing resources

You will see these used a lot for setting up class properties, like database information or generic values

Make sure to mention other magic methods __call, __callStatic, __get, __set, __isset, __unset, __sleep, __wakeup, __toString, __invoke, __set_state, __clone

__call - triggered when invoking inaccessible methods in an object context

__callStatic - triggered when invoking inaccessible methods in a static context

__get - used for reading data from inaccessible properties

__set - used when writing data to inaccessible properties

__isset - triggered by calling isset or empty on inaccessible properties

__unset - triggered when calling unset on an inaccessible property

__sleep - triggered when calling serialize

__wakeup - triggered when calling unserialize

__toString - allows the class to decide how to react when it is treated like a string

__invoke - triggered when a script tries to call an object as a function

__set_state - static method trigged by var_export

__clone - triggered when an object copy is created by using the clone keyword
